# AI Usage Summary

## AI Tool Used
- Codex

## How I Used AI
- Planning feature work before coding.
- Implementing React components and context/state updates.
- Debugging issues and addressing review feedback.
- Preparing demo data and documentation.
- I used different branches whenever implementing with AI so that the codebase stayed clean and I could test the AI implemented features before merging with the main branch.
- I also used the code review tool from Codex to review the branches before I merged them, letting the AI check its own code for security issues and possible bugs.

## What AI Helped Build
- Streak and weekly goal tracking.
- Mood notes (create, view, and edit).
- Dark mode toggle and chart theme updates.
- Mood calendar heatmap.
- Sleep tracker and sleep insights.
- Demo import dataset for presentation.

## Limitations and Issues
- The main challenge was clearly communicating what I wanted so AI understood the exact scope.
- Some prompts needed clarification before outputs matched expectations.

## Human Review and Corrections
- I kept changes small and simple when AI suggested broader edits.
- I checked outputs against project requirements in `README.md` and `AGENTS.md`.
- I used build checks and manual review before finalizing work.
